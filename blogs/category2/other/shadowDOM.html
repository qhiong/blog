    shadowDOM

    2/15/2022 tag4

    # 概念

    Web components 的一个重要属性是封装——可以将标记结构、样式和行为隐藏起来，并与页面上的其他代码相隔离，保证不同的部分不会混在一起，可使代码更加干净、整洁。其中，Shadow DOM 接口是关键所在，它可以将一个隐藏的、独立的 DOM 附加到一个元素上。

    • Custom elements（自定义元素）： 一组JavaScript API，允许您定义custom elements及其行为，然后可以在您的用户界面中按照需要使用它们。
    • Shadow DOM（影子DOM）： 一组JavaScript API，用于将封装的“影子”DOM树附加到元素（与主文档DOM分开呈现）并控制其关联的功能。通过这种方式，可以保持元素的功能私有，这样它们就可以被脚本化和样式化，而不用担心与文档的其他部分发生冲突。
    • HTML templates（HTML模板）： < template >和 < slot >元素使您可以编写不在呈现页面中显示的标记模板。然后它们可以作为自定义元素结构的基础被多次重用。

    # 作用

    Shadow-dom 是游离在 DOM 树之外的节点树，但是他的创建基于普通 DOM 元素，并且创建后的 Shadow-dom 节点可以从界面上直观的看到。更重要的是，Shadow-dom 具有良好的密封性。

    Shadow host：一个常规 DOM节点，Shadow DOM 会被附加到这个节点上。

    Shadow tree：Shadow DOM内部的DOM树。

    Shadow boundary：Shadow DOM结束的地方，也是常规 DOM开始的地方。

    Shadow root: Shadow tree的根节点

    # 支持shadowDOM元素的标签(18个)

    article aside blockquote body div footer h1-h6 header main nav p section span

    # 使用API

    1. 语法

      var shadowroot = element.attachShadow(shadowRootInit);

    2. 参数

      shadowRootInit

      mode 模式 : 指定 Shadow DOM 树封装模式的字符串

       open : shadow root元素可以从js外部访问根节点

      element.shadowRoot; // 返回一个ShadowRoot对象

       closed 拒绝从js外部访问关闭的shadow root节点

      element.shadowRoot; // 返回null

    3. 返回值

      返回一个 ShadowRoot 对象或者 null。

    4. 异常 说明

      InvalidStateError(无效状态错误): 添加的元素已经是一个shadow host（影子主机）

      NotSupportedError(不被支持错误): 应该添加 HTML 元素的命名空间之外的shadow root, 或者这个元素不能有其他shadow挂载到它上面 .

    # focus-within

    focus-within 是一个CSS 伪类 ，表示一个元素获得焦点，或该元素的后代元素获得焦点。也就是说即便是Shadow DOM，当其获得焦点时，也会使得外部的父级DOM元素上的focus-within生效。

    # 特点

    # 1.普通DOM元素失效

    当给一个普通元素附加Shadow DOM后，其原本所包含的所有普通DOM元素都会失效，一个普通元素下不会同时存在Shadow DOM 和 普通 DOM。虽然仍然可以使用js获取到这部分普通元素，但是浏览器不会将其渲染到页面中。

    # 2.样式隔离

    1. 外部CSS（包含style和外链资源）中被应用在Shadow Host元素的样式中（包括继承来的样式）能被继承的属性都会被Shadow DOM中的元素继承，通过选择器无法直接对Shadow DOM的元素应用样式。
    2. Shadow DOM中style或者引入的CSS不会对外部元素造成影响

    # 3.事件委托

    如果在 Shadow DOM 的外部采用了事件委托机制，则无法通过Event.target准确的判断shadowDOM内部目标事件元素
